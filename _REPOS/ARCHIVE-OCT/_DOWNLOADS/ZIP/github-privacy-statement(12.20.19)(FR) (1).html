Déclaration de confidentialité GitHub
Les informations que GitHub collecte
« Les Informations personnelles d’utilisateur » correspondent à toutes les informations concernant un Utilisateur pouvant, seules ou
associées à d’autres informations, l’identifier personnellement ou être raisonnablement liées à lui. Les informations telles qu’un nom
d’utilisateur et un mot de passe, une adresse e-mail, un vrai nom, une adresse Internet Protocol (IP) ou une photo sont des exemples
d’« Informations personnelles d’utilisateur ».
Les Informations personnelles d’utilisateur n’incluent pas les informations agrégées et ne permettant pas l’identification personnelle
qui n’identifient pas l’Utilisateur ou ne peuvent pas être raisonnablement liées à lui. Nous pouvons utiliser de telles informations à des
fins de recherche et pour exploiter, analyser, améliorer et optimiser notre Site Web et notre Service.
Informations directement fournies par les utilisateurs à GitHub
Informationsd’enregistrement
Au moment de la création du compte, nous demandons certaines informations basiques. Lorsque vous créez votre nom d’utilisateur et
votre mot de passe, nous vous demandons une adresse e-mail valide.
Informationsdepaiement
Si vous souscrivez à un Compte payant chez nous, envoyez des fonds via le Programme de sponsor GitHub ou achetez une
application sur le Marché GitHub, nous collectons votre nom complet, votre adresse et vos informations de carte de crédit ou PayPal.
Veuillez noter que GitHub ne traite ni ne stocke vos informations de carte de crédit ou PayPal, mais notre processeur de paiement
tiers, oui.
Si vous mettez en vente une application sur le Marché GitHub, nous exigerons vos informations bancaires. Si vous collectez des fonds
via le Programme de sponsor GitHub, nous exigerons des informations supplémentaires par le biais du processus d’enregistrement
afin que vous puissiez participer et recevoir des fonds via ces services.
Informationsdeprofil
Vous pouvez choisir de nous donner plus d’informations, comme votre nom complet, un avatar pouvant inclure une photo, votre
biographie, votre emplacement, votre entreprise et une URL vers un site Web tiers. Ces informations peuvent inclure des Informations
personnelles d’utilisateur. Veuillez noter que les informations de votre profil peuvent être visibles par d’autres Utilisateurs de notre
Service.
Informations automatiquement collectées par GitHub à partir de votre utilisation du Service
Informationstransactionnelles
Si vous avez un compte payant chez nous, vendez une application répertoriée sur le Marché GitHub ou collectez des fonds par le biais
du Programme de sponsor GitHub, nous collectons automatiquement certaines informations sur vos transactions sur le Service, telles
que la date, l’heure et le montant facturé.
Informationsd’utilisation
Si vous accédez à notre Service ou à notre Site Web, nous collectons automatiquement les mêmes informations basiques que la
plupart des autres services, sous réserve, le cas échéant, de votre consentement. Cela inclut les informations sur votre utilisation du
Service, comme les pages que vous consultez, le site référent, votre adresse IP et vos informations de session ainsi que la date et
l’heure de chaque requête. Il s’agit d’informations que nous collectons auprès de tous les visiteurs du site Web, qu’ils aient un Compte
ou non. Ces informations peuvent inclure des Informations personnelles d’utilisateur.
Informationsdecookiesettechnologiessimilaires
Comme décrit plus en détail ci-dessous et sous réserve, le cas échéant, de votre consentement, nous collectons automatiquement
des informations des cookies et technologies similaires (paramètres et ID de cookie) pour vous maintenir connecté, enregistrer vos
préférences et vous identifier, vous et votre périphérique.
Informationsdedispositif
Nous pouvons collecter certaines informations sur votre dispositif, comme son adresse IP, son navigateur ou des informations
d’application client, des préférences de langue, le système d’exploitation et la version d’application, l’ID et le type de dispositif ainsi
que le fabricant et le modèle du dispositif. Ces informations peuvent inclure des Informations personnelles d’utilisateur.
Informations collectées auprès de tiers
GitHub peut collecter des Informations personnelles d’utilisateur auprès de tiers. Par exemple, cela peut arriver si vous vous inscrivez
à une formation ou pour recevoir des informations sur GitHub auprès de l’un de nos fournisseurs, partenaires ou affiliés. GitHub
n’achète pas d’Informations personnelles d’utilisateur auprès de courtiers en information.
Les informations que GitHub ne collecte pas
Nous ne collectons pas intentionnellement des « Informations personnelles sensibles », telles que les données personnelles révélant
l’origine ethnique ou raciale, les opinions politiques, les croyances religieuses ou philosophiques ou l’appartenance syndicale, ainsi
que le traitement des données génétiques, les données biométriques aux fins de l’identification unique d’une personne physique, les
données concernant la santé ou les données concernant l’orientation sexuelle ou la vie sexuelle d’une personne physique. Si vous
choisissez de stocker des Informations personnelles sensibles sur nos serveurs, vous devez vous conformer à tout contrôle
réglementaire concernant ces données.
Si vous êtes un enfant de moins de 13 ans, nous n’avez pas le droit d’avoir un Compte sur GitHub. GitHub ne collecte pas sciemment
d’informations auprès d’enfants de moins de 13 ans ni n’adresse directement aucun contenu à des enfants de moins de 13 ans. Si
nous découvrons ou avons des raisons de penser que vous avez moins de 13 ans, nous devrons fermer votre Compte. Nous ne
voulons pas vous décourager d’apprendre à coder, mais ce sont les règles. Pour plus d’informations sur la fermeture du compte,
veuillez consulter nos Conditions générales d’utilisation. La limite d’âge minimum peut varier selon les pays et si vous n’avez pas l’âge
minimum de consentement à la collecte de données dans votre pays, vous ne pouvez pas avoir un Compte sur GitHub.
Nous ne collectons pas intentionnellement d’Informations personnelles d’utilisateur stockées dans vos dépôts ou d’autres entrées de
contenu de forme libre. Les informations personnelles contenues dans un dépôt sont de la responsabilité du propriétaire du dépôt.

Nous utilisons vos Informations personnelles d’utilisateur pour répondre aux demandes d’assistance.
Nous utilisons vos Informations personnelles d’utilisateur et d’autres données pour vous faire des recommandations, par exemple
vous proposer des projets que vous souhaiteriez suivre ou auxquels vous souhaiteriez contribuer. Nous apprenons de votre

Nous pouvons utiliser vos Informations personnelles d’utilisateur pour vous inviter à participer à des sondages, à des programmes
bêta ou d’autres projets de recherche sous réserve, le cas échéant, de votre consentement.
Nous utilisons les Informations d’utilisation et les Informations de dispositif pour mieux comprendre comment nos utilisateurs
utilisent GitHub et pour améliorer notre Site Web et notre Service.
Nous pouvons utiliser vos Informations personnelles d’utilisateur si cela est nécessaire pour des raisons de sécurité ou en cas de
fraude présumée ou de tentatives de nuire à GitHub ou à ses Utilisateurs.
Nous pouvons utiliser vos Informations personnelles d’utilisateur pour nous conformer à nos obligations légales, protéger notre
propriété intellectuelle et appliquer nos Conditions générales d’utilisation.
Nous limitons notre utilisation de vos Informations personnelles d’utilisateur aux fins listées dans cette Déclaration de
confidentialité. Si nous avons besoin d’utiliser vos Informations personnelles d’utilisateur à d’autres fins, nous vous demanderons
d’abord l’autorisation. Vous pouvez toujours consulter les informations dont nous disposons, comment nous les utilisons et les
autorisations que vous nous avez accordées dans votre profil d’utilisateur.
Nos bases juridiques en matière de traitement des informations
Dans la mesure où notre traitement de vos Informations personnelles d’utilisateur est soumis à certaines lois internationales (y
compris, sans s’y limiter, au Règlement général de l’Union européenne sur la protection des données [RGPD]), GitHub est tenu de
vous informer de la base juridique sur laquelle nous traitons les Informations personnelles d’utilisateur. GitHub traite les Informations
personnelles d’utilisateur selon les bases juridiques suivantes :
Exécution du contrat :
Lorsque vous créez un Compte GitHub, vous fournissez vos Informations d’enregistrement. Nous avons besoin de ces
informations pour que vous acceptiez nos Conditions générales d’utilisation et nous traitons ces informations sur la base de
l’exécution de ce contrat. Nous traitons également votre nom d’utilisateur et votre adresse e-mail sur des bases juridiques
différentes, comme décrit ci-dessous.
Si vous disposez d’un compte payant chez nous, nous collectons et traitons des Informations de paiement supplémentaires
sur la base de l’exécution de ce contrat.
Lorsque vous achetez ou vendez une application répertoriée sur notre Marché ou lorsque vous envoyez ou recevez des fonds
via le Programme de sponsor GitHub, nous traitons des Informations de paiement et d’autres éléments afin d’exécuter le
contrat s’appliquant à ces services.
Consentement :
Nous comptons sur votre consentement pour utiliser vos Informations personnelles d’utilisateur dans les cas suivants :
lorsque vous fournissez des renseignements sur votre profil d’utilisateur ; lorsque vous décidez de participer à une formation
GitHub, à un projet de recherche, à un programme bêta ou à un sondage ; et à des fins de marketing, le cas échéant. Toutes
ces Informations personnelles d’utilisateur sont entièrement facultatives et vous avez la possibilité d’y accéder, de les
modifier et de les supprimer à tout moment. Vous ne pouvez pas supprimer votre adresse e-mail, mais vous pouvez la rendre
privée. Vous pouvez retirer votre consentement à tout moment.
Intérêts légitimes :
En général, le reste du traitement des Informations personnelles d’utilisateur est nécessaire aux fins de notre intérêt légitime,
par exemple, à des fins de conformité juridique ou de sécurité ou pour assurer la confidentialité, l’intégrité, la disponibilité et
la résilience constantes du Service, du Site Web et des systèmes de GitHub.
Si vous souhaitez demander la suppression des données traitées sur la base de votre consentement ou si vous vous opposez au
traitement de vos informations personnelles, veuillez utiliser notre formulaire de contact de confidentialité.
Le partage des informations que nous collectons
Nous pouvons partager vos Informations personnelles d’utilisateur avec des tiers dans l’un des cas suivants :
Avec votre consentement. Nous partageons vos Informations personnelles d’utilisateur, si vous y consentez, après vous avoir indiqué
les informations qui seront partagées, avec qui et pourquoi. Par exemple, si vous achetez une application répertoriée sur notre
Marché, nous partageons votre nom d’utilisateur avec le Développeur de l’application pour lui permettre de vous fournir des services.
De plus, par le biais de vos actions sur GitHub, vous pouvez nous indiquer votre volonté de partager vos Informations personnelles
d’utilisateur. Par exemple, si vous rejoignez une Organisation, vous indiquez votre volonté de fournir au propriétaire de l’Organisation
la possibilité de visualiser votre activité dans le journal d’accès de l’Organisation.
Avec les fournisseurs de service. Nous partageons vos Informations personnelles d’utilisateur avec un nombre limité de fournisseurs
de service qui les traitent en notre nom pour fournir ou améliorer notre Service et qui ont accepté des restrictions de confidentialité
semblables à celles de notre Déclaration de confidentialité en signant des accords de protection des données ou en prenant des
engagements similaires. Nos fournisseurs de service procèdent à la gestion des paiements, à la gestion de l’assistance clientèle, à la
transmission de données de réseau, à la sécurité et à d’autres services similaires. Tandis que GitHub traite toutes les Informations
personnelles d’utilisateur aux États-Unis, nos fournisseurs de service peuvent traiter des données en dehors des États-Unis ou de
l’Union européenne. Si vous souhaitez savoir qui sont nos fournisseurs de service, veuillez consulter notre page Sous-traitants.
À des fins de sécurité. Si vous êtes membres d’une Organisation, GitHub peut partager votre nom d’utilisateur, vos Informations
d’utilisation et vos Informations de dispositif associés à cette Organisation avec un propriétaire et/ou un administrateur de
l’Organisation ayant accepté les Conditions d’utilisation ou les accords client en vigueur, dans la mesure où ces informations ne sont
fournies qu’à des fins d’enquête ou de réponse à un incident de sécurité affectant ou compromettant la sécurité de cette Organisation.
À des fins de divulgation légale. GitHub veille à assurer la transparence en respectant les procédures et les obligations légales. À
moins que la loi ou qu’une décision de justice ne s’y oppose ou sauf dans de rares et exigeantes circonstances, nous déployons des
efforts raisonnables pour informer les utilisateurs lorsque nous sommes légalement contraints de divulguer leurs informations. GitHub
peut divulguer les Informations personnelles d’utilisateur ou toutes autres informations que nous collectons sur vous aux forces de
l’ordre si nous y sommes contraints par une assignation à comparaître, un mandat de recherche ou toute autre décision
gouvernementale similaire ou si nous pensons de bonne foi que cette divulgation est nécessaire au respect de nos obligations légales,
pour protéger notre propriété ou nos droits ou ceux de tiers ou du public en général.
Pour plus d’informations sur la divulgation en réponse à des exigences juridiques, consultez nos Directives en cas de demandes
juridiques de données d’utilisateur.
Changement de direction ou vente. Nous pouvons partager des Informations personnelles d’utilisateur si nous sommes parties
prenantes d’une fusion, d’une vente ou d’une acquisition d’entités morales ou d’unités commerciales. Si un tel transfert de propriété
survient, nous nous assurerons qu’il est fait dans des conditions qui préservent la confidentialité des Informations personnelles
d’utilisateur et vous serez informés sur notre Site Web ou par e-mail avant tout transfert de vos Informations personnelles d’utilisateur.
L’organisation recevant des Informations personnelles d’utilisateur devra honorer toutes les promesses indiquées dans notre
Déclaration de confidentialité ou nos Conditions générales d’utilisation.
Informations agrégées et ne permettant pas l’identification personnelle. Nous partageons certaines des informations agrégées et ne
permettant pas l’identification personnelle avec des tiers sur la façon dont nos utilisateurs, collectivement, utilisent GitHub ou dont nos
utilisateurs répondent à nos autres offres, telles que nos conférences ou événements. Par exemple, nous pouvons compiler des
statistiques sur l’activité Open Source de GitHub.
Nous ne vendons pas vos Informations personnelles d’utilisateur en échange d’une contrepartie financière ou autre.
Veuillez noter que : la loi sur la protection du consommateur (California Consumer Privacy Act of 2018, « CCPA ») exige des
entreprises qu’elles stipulent dans leur politique de confidentialité si elles divulguent ou non des informations personnelles en échange
d’une contrepartie financière ou autre. Bien que la CCPA ne couvre que les habitants de Californie, quand elle entrera en vigueur, nous
étendrons volontairement ses droits fondamentaux aux utilisateurs du contrôle de leurs données à tous nos utilisateurs vivant aux
États-Unis, et pas uniquement ceux vivant en Californie. Pour en savoir plus sur la CCPA et comment nous nous y conformons,
cliquez ici.
Autres informations importantes
Contenu du dépôt
Les employés GitHub n’ont pas accès aux dépôts privés sauf pour des raisons de sécurité, pour aider le propriétaire du dépôt à des
fins d’assistance, pour maintenir l’intégrité du Service ou pour respecter nos obligations légales. Toutefois, bien que nous ne
recherchions généralement pas de contenu dans vos dépôts, nous pouvons scanner nos serveurs et nos contenus pour repérer des
jetons ou des signatures de sécurité, des malwares actifs connus, ou des images liées à l’exploitation des enfants. Pour plus
d’informations, consultez nos Conditions générales d’utilisation.
Si votre dépôt est public, n’importe qui peut consulter son contenu. Si vous intégrez des Informations personnelles sensibles,
confidentielles ou privées, telles que des adresses e-mail ou des mots de passe, dans vos dépôts publics, ces informations peuvent
être indexées par les moteurs de recherche ou utilisées par des tiers.
Pour en savoir plus sur les informations personnelles dans les dépôts publics, cliquez ici.
Informations publiques sur GitHub
La plupart des services et fonctionnalités de GitHub sont accessibles au public. Si votre contenu est accessible au public, des tiers
peuvent y avoir accès et l’utiliser conformément à nos Conditions d’utilisation, par exemple, en consultant votre profil ou vos dépôts
ou en extrayant des données via notre API. Nous ne vendons pas ce contenu ; c’est le vôtre. Toutefois, nous autorisons des tiers, tels
que des organismes de recherche ou des archives, à compiler les informations GitHub accessibles au public. D’autres tiers, tels que
des courtiers en information, sont connus pour pratiquer le data scraping sur GitHub et compiler également des données.
Vos Informations personnelles d’utilisateur associées à votre contenu peuvent être collectées par des tiers dans le cadre de ces
compilations de données GitHub. Si vous ne souhaitez pas que vos Informations personnelles d’utilisateur apparaissent dans les
compilations de données GitHub de tiers, veuillez ne pas rendre ces informations publiques et assurez-vous de configurer votre
adresse e-mail pour qu’elle apparaisse en privé dans votre profil d’utilisateur et dans vos paramètres de validation Git. L’adresse e-
mail de nos Utilisateurs est actuellement définie comme privée par défaut, mais les Utilisateurs GitHub existants devront peut-être
mettre à jour leurs paramètres.
Si vous souhaitez compiler des données GitHub, vous devez respecter nos Conditions générales d’utilisation concernant le scraping et
la confidentialité et vous ne pouvez utiliser que les Informations personnelles d’utilisateur accessibles au public que vous avez
rassemblées aux fins autorisées par notre utilisateur. Par exemple, lorsqu’un utilisateur GitHub a rendu une adresse e-mail publique à
des fins d’identification et d’attribution, vous ne pouvez pas utiliser cette adresse e-mail à des fins commerciales et publicitaires. Nous
attendons de vous que vous protégiez de façon raisonnable les Informations personnelles d’utilisateur que vous avez collectées sur
GitHub et que vous répondiez rapidement aux plaintes, demandes de suppression et demandes « Ne pas contacter » de la part de
GitHub et des utilisateurs de GitHub.
De la même façon, les projets sur GitHub peuvent inclure des Informations personnelles d’utilisateur publiquement disponibles
collectées dans le cadre du processus collaboratif. Si vous avez une plainte concernant les Informations personnelles d’utilisateur sur
GitHub, veuillez consulter notre section Règlement des plaintes.

Déclaration de confidentialité, et nous collectons la quantité d’Informations personnelles d’utilisateur nécessaires et ne les utilisons
qu’aux fins que vous nous avez accordées.
Accès et contrôle aux informations que nous collectons
Si vous êtes déjà un utilisateur GitHub, vous pouvez accéder à vos informations de profil d’utilisateur basiques, les mettre à jour, les
modifier ou les supprimer en modifiant votre profil d’utilisateur ou en contactant l’Assistance GitHub ou l’Assistance Premium GitHub.
Vous pouvez contrôler les informations que nous collectons sur vous en limitant les informations apparaissant sur votre profil, en
maintenant vos informations à jour ou en contactant l’Assistance GitHub or l’Assistance Premium GitHub.
Si GitHub traite des informations sur vous, telles que les informations que GitHub reçoit de tiers et que vous n’avez pas de compte,
vous pouvez, sous réserve des lois en vigueur, y accéder, les mettre à jour, les modifier, les supprimer ou vous opposer à leur
traitement en contactant l’Assistance GitHub or l’Assistance Premium GitHub.
Portabilité des données
En tant qu’Utilisateur GitHub, vous pouvez toujours emporter vos données avec vous. Vous pouvez, par exemple, cloner vos dépôts
sur votre bureau ou utiliser nos outils de portabilité des données pour télécharger les informations que nous avons sur vous.
Rétention et suppression des données
En général, GitHub conserve les Informations personnelles d’utilisateur tant que votre compte est actif ou tant qu’elles sont
nécessaires pour vous fournir des services.
Si vous souhaitez annuler votre compte ou supprimer vos Informations personnelles d’utilisateur, vous pouvez le faire dans votre profil
d’utilisateur. Nous conservons et utilisons vos informations afin de respecter nos obligations légales, régler des différends et faire
appliquer nos conditions, mais nous nous engageons, sauf obligation légale, à supprimer votre profil dans son intégralité (sans motif)
dans les 90 jours suivant votre demande. Vous pouvez contacter l’Assistance GitHub ou l’Assistance Premium GitHub pour demander
la suppression des données traitées sur la base du consentement dans les 30 jours.
Une fois un compte supprimé, certaines données, telles que les contributions apportées aux dépôts d’autres Utilisateurs et les
commentaires postés sur les sujets d’autres Utilisateurs, seront conservées. Toutefois, nous supprimerons ou anonymiserons vos
Informations personnelles d’utilisateur, y compris votre nom d’utilisateur et votre adresse e-mail, du champ auteur des sujets,
demandes d’acquisition et commentaires en les associant à un utilisateur fantôme.
Cela étant dit, l’adresse e-mail que vous avez fournie via vos paramètres de validation Git sera toujours associée à vos validations
dans le système Git. Si vous choisissez de rendre votre adresse e-mail privée, vous devez également mettre à jour vos paramètres de
validation Git. Nous ne pouvons ni modifier ni supprimer les données de l’historique des validations Git (le logiciel Git est conçu pour
tenir un registre), mais nous vous permettons de contrôler les informations que vous mettez dans ce registre.
Notre utilisation des cookies et des mouchards
Cookies
GitHub utilise des cookies pour faciliter les interactions avec notre service et les rendre pertinentes. Les cookies sont de petits fichiers
textes que les sites Web stockent souvent sur les disques durs ou les périphériques mobiles des visiteurs. Nous utilisons des cookies
(et des technologies similaires comme le HTML5 localStorage) pour vous maintenir connecté, enregistrer vos préférences et vous
fournir des informations pour le développement futur de GitHub. Par mesure de sécurité, nous utilisons des cookies pour identifier un
appareil. En utilisant notre Site Web, vous acceptez que ces cookies soient placés sur votre ordinateur ou appareil. Si vous désactivez
les cookies sur votre navigateur ou votre appareil, vous ne pourrez plus vous connecter ni utiliser les services de GitHub.
Nous avons créé une page Web sur les cookies et les mouchards décrivant les cookies utilisés, la nécessité que nous avons à utiliser
ces cookies et le type de cookies dont il s’agit (temporaires ou permanents). Elle répertorie également les fournisseurs d’analyse tiers
et les autres fournisseurs de services et détaille les parties du site Web que nous les autorisons à analyser.
Mouchards et analyses
Nous avons recours à un certain nombre de fournisseurs de service et d’analyse tiers pour nous aider à évaluer l’utilisation de GitHub
par les Utilisateurs, compiler des rapports statistiques sur l’activité et améliorer votre contenu et les performances du Site Web. Nous
n’utilisons que ces fournisseurs d’analyse tiers que sur certaines parties de notre site Web et tous ont signé un accord sur la
protection des données limitant le type d’Informations personnelles d’utilisateur qu’ils peuvent collecter et le but dans lequel ils
peuvent traiter ces informations. De plus, nous utilisons notre propre logiciel d’analyse interne pour fournir des fonctionnalités et
améliorer notre contenu et nos performances.
Certains navigateurs ont intégré un en-tête « Do Not Track » (DNT) permettant de signaler aux sites Web que vous visitez que vous ne
souhaitez pas être suivi. GitHub répond aux signaux DNT et respecte la norme W3C en matière de réponse aux signaux DNT. Si vous
n’avez pas activé le signal DNT sur un navigateur qui le prend en charge, les cookies de certaines parties de notre Site Web finiront
par suivre votre activité de navigation sur d’autres services en ligne, bien que nous ne permettions pas aux tiers autres que nos
fournisseurs de service et d’analyse de suivre l’activité de nos Utilisateurs sur GitHub dans le temps.
Protection de vos informations par GitHub
GitHub prend toutes les mesures raisonnablement nécessaires pour protéger les Informations personnelles d’utilisateur d’une
destruction, d’une altération ou d’un accès non autorisés, maintenir l’intégrité des données et assurer l’utilisation appropriée des
Informations personnelles d’utilisateur.
GitHub met en œuvre un programme de sécurité de l’information. Notre programme :
s’aligne aux cadres reconnus du secteur ;
intègre des mesures de sécurité raisonnablement conçues pour protéger la confidentialité, l’intégrité, la disponibilité et la
résilience des données de nos Utilisateurs ;
est approprié à la nature, à la taille et à la complexité des opérations commerciales de GitHub ;
inclut des processus de réponse aux incidents et de notification de violation des données ; et
se conforme aux lois et réglementations relatives à la sécurité informatique en vigueur dans les zones géographiques dans
lesquelles GitHub exerce ses activités.
Dans le cas où une violation de données affecterait vos Informations personnelles d’utilisateur, nous nous engageons à intervenir
rapidement pour limiter l’impact de la violation et à informer dans les meilleurs délais les Utilisateurs affectés.
La transmission de données sur GitHub est chiffrée à l’aide des protocoles SSH et HTTPS (TLS) et le contenu du dépôt Git est chiffré
au repos. Nous gérons nos cages et nos baies dans de gros centres de données dotés d’un haut niveau de sécurité physique et
réseau, et lorsque les données sont stockées via un fournisseur de stockage tiers, elles sont chiffrées.
Aucune méthode de transmission ou de stockage électronique n’est 100 % sûre. C’est pourquoi nous ne pouvons pas garantir une
sécurité absolue. Pour plus d’informations, veuillez consulter nos communications en matière de sécurité.
Pratiques internationales de confidentialité de GitHub
Nous stockons et traitons les informations que nous collectons aux États-Unis conformément à cette Déclaration de confidentialité,
bien que nos fournisseurs de service puissent stocker et traiter des données en dehors des États-Unis. Nous sommes toutefois
conscients que certains de nos clients viennent de régions et de pays différents et ont des attentes différentes en matière de
confidentialité. Nous nous efforçons de répondre à ces besoins, même lorsque les États-Unis n’ont pas le même cadre que les autres
pays.
Nous offrons à nos clients du monde entier des normes élevées en matière de protection de la vie privée (tel que décrit dans cette
Déclaration de confidentialité), quel que soit leur pays d’origine ou celui dans lequel ils habitent et nous sommes fiers des niveaux de
notification, de choix, de responsabilité, de sécurité, d’intégrité des données, d’accès et de recours que nous offrons. Nous faisons
tout notre possible pour respecter les lois en vigueur en matière de confidentialité des données là où nous exerçons nos activités.
Nous travaillons avec notre Responsable de la protection des données au sein d’une équipe interfonctionnelle qui supervise les efforts
de confidentialité des données déployés. De plus, si nos fournisseurs ou affiliés ont accès à des Informations personnelles
d’utilisateur, ils doivent signer des accords les obligeant à se conformer à nos politiques de confidentialité et aux lois en vigueur en
matière de confidentialité des données.

Comment nous communiquons avec vous
Nous utilisons votre adresse e-mail pour communiquer avec vous, si vous êtes d’accord, et uniquement pour les raisons avec
lesquelles vous êtes d’accord. Par exemple, si vous contactez notre équipe d’assistance avec une demande, vous vous répondrons
par e-mail. Vous avez beaucoup de contrôle sur l’utilisation et le partage de votre adresse e-mail sur et via GitHub. Vous pouvez gérer
vos préférences de communication sur votre profil d’utilisateur.
Par nature, le système de contrôle de version Git associe de nombreuses actions à l’adresse e-mail d’un Utilisateur, par exemple, les
messages de validation. Nous ne pouvons pas modifier la plupart des aspects du système Git. Si vous souhaitez que votre adresse e-
mail reste privée, même lorsque vous publiez des commentaires sur les dépôts publics, vous pouvez créer une adresse e-mail privée
dans votre profil d’utilisateur. Vous devez également mettre à jour votre configuration locale Git pour utiliser votre adresse e-mail
privée. Cela ne changera pas la façon dont nous communiquons avec vous, mais impactera la façon dont les autres vous voient.
L’adresse e-mail de nos Utilisateurs est actuellement définie comme privée par défaut, mais les utilisateurs GitHub existants devront
peut-être mettre à jour leurs paramètres. Pour plus d’informations sur les adresses e-mail dans les messages de validation, cliquez ici.
Selon vos paramètres d’e-mail, GitHub peut occasionnellement envoyer des e-mails de notification sur des modifications affectant un
dépôt que vous observez, de nouvelles fonctionnalités, des demandes de feedback, d’importantes modifications de politique ou pour
offrir une assistance à la clientèle. Nous envoyons également des e-mails marketing basés sur vos choix et conformément aux lois et
règlements en vigueur. Vous trouverez un lien « Se désinscrire » au bas de chaque e-mail marketing que nous vous envoyons. Veuillez
noter que vous ne pouvez pas refuser de recevoir des communications importantes de notre part, telles que les e-mails de notre
équipe d’assistance ou les e-mails du système, mais vous pouvez configurer vos paramètres de configuration sur votre profil pour
vous retirer des autres communications.
Nos e-mails peuvent contenir un pixel espion, qui est une petite image transparente qui nous indique si vous avez ouvert ou non l’e-
mail et quelle est votre adresse IP. Nous utilisons ce pixel espion pour rendre nos e-mails plus efficaces et nous assurer que nous ne
vous envoyons pas d’e-mails indésirables.

Dans l’éventualité où un différend surviendrait entre vous et GitHub concernant la gestion de vos Informations personnelles
